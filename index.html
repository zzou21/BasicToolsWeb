Welcome to Basic Tools

Current version: 1.3.37

Basic Tools currently has x tools, where x is the middle value (minor version #) in the version number, e.g., version 0.x.00.

An open-source and minimum-computing suite of everyday computer utilities that runs locally on your machines.

Current Tools Available:

  1. File Finder: Coveniently searchs and opens files within a specific folder.
  2. Image Converter: Converts images from HEIC/HEIF, JPG, PNG, and other common image formats into other image formats or PDF and rename each one as you go.
  3. PDF Merger: Merges two or more PDFs into one document, with the option of rearranging the order of the PDFs.

More tools to be added. Suggest a new tool via the "Forum" page.

I use Mac but not sure whether I need the Apple Silicon or Intel version?

There are two types of Mac computers, delineated by which processor they use. One uses Apple Silicon and the other Intel. You have to download the Basic Tools that matches your Mac's processor.

To determine which Mac processor you have:

Step 1: Go to the  icon on the top left corner of your MacBook. Then, in the droptown list, click on "About This Mac".



Step 2: After clicking on "About This Mac" you should see a window that looks something like this:

This image shows a MacBook that has an Apple Silicon chip. You can tell because it says "Apple M4". You might have an M1, M2, M3, or another Mx number. If it says "Apple" and has "M" + a number, it is an Apple Silicon chip.

If, instead, your screen shows you something like this:

Then it means you have an Intel chip. The exact wording on your screen might not be the same as the image above, but if it has the word "Intel" in it somewhere, then you have an Intel MacBook.

After determining which processor your MacBook has, you can return to the home screen to download the Mac version of Basic Tools that fits you.

Mac Download and Post-Download Clean Up

The .dmg File

For Mac computers, the way to download an app from the internet (and not from the App Store) is that the app provider (me) lets you download a .dmg file, which is Apple's proprietary disc format to provide security during software distribution. When you click on your .dmg file to download Basic Tools, the .dmg file will create a virtual disk (which you can imagine it as metaphorically plugging in a USB drive), and both will appear and remain on your Desktop (or wherever on your computer that you downloaded them to). After the installation process, you can delete them both.

When you first download Basic Tools, you should see something like this showing up in the directory that you have downloaded it to. It is a file with a .dmg suffix:

After you've clicked on the above .dmg file, it should have created a disk that looks like this in your directory:

Clicking on the disk above should have opened a window that prompted you to drag the Basic Tools icon into your "Applications" folder. Doing so will complete the installation process.

After installing Basic Tools, you can eject the disk by right clicking on its icon:

You can also delete the .dmg file by dragging its icon into Trash.

Windows users: please read this download instruction before downloading the app

Windows Download Instructions

If you are familiar with downloading a Windows app via a .zip file, unzipping it, finding the Application .exe file in it, creating a shortcut for it on your Desktop, and not deleting the unzipped folder, then you do not need to read this page.

Why This?

I'm adding this instruction page for Windows users is that Basic Tools hasn't been code signed (what is "code sign"?) with Windows yet, meaning, if you download the app directly, it will trigger your Windows firewall and reject the app. This is why when you download the app on the homescreen, it gives you a .zip file that contains the app (along with a bunch of other files that we don't need to worry about).

The reason why Basic Tools hasn't been code signed yet (which can make the downloading process much smoother and this instruction page unnecessary) is that it costs a lot of money to code sign with Windows. The cheapest one I found is $129 a year. Capitalists (same with Apple). I will, of course, code sign it as soon as I find enough funding to do so to ensure further security and better user experience. The cost behind code signing is one (sometimes major) hidden cost in software development that is a pain for open-source and non-commercial softwares. If you are interested in supporting Basic Tools, please (please) reach out to me on the "about" page (thanks!).

To smoothly install Basic Tools on Windows:

Step 1:

When you click on the Windows download button on the Basic Tools home screen, you should've received a .zip file in your "Download" folder (or whichever folder that you set your Windows machine to receive downloads to). Right click on it to unzip it (some machines might call this process "Extract All"). You can tell it is a zipped file because it should sa "Compressed (zipped) Folder" under the "Type" column:

Step 2:

After unzipping (or extracting) the zip folder, you should see a folder of the same name (but with the "Type" just saying "Folder" and not "Compressed (zipped) Folder"). Enter that unzipped folder.

Step 3:

After you enter the unzipped folder, you should see quite a few files. We do not need to worry about most of them. You just need to find the one that says "BasicTools," with Basic Tools' icon, and has the word "Application" under the document "Type" column:

Step 4:

Right click on the BasicTools application file to show the different options. For some machine, you will see a "Send to" button right away; for some, you need to click on "Show more options" to get to it:

Step 5:

Click on "Show more options" to find "Send to", and then click on the option "Desktop (create shortcut)":

Step 6:

After clicking on "Desktop (create shortcut)," you should see the Basic Tools icon showing up on your desktop. Now, in the future, you can easily access it from there. But, if this is your first time opening Basic Tools or that you have downloaded a new version of it, there's an extra step:

Step 7:

The first time you open Basic Tools, your Windows machine will give you a blue warning window saying: "Windows protected your PC" (because I didn't pay over a hundred dollars for code signing). You can click on "More info" and then "Run Anyway." Yuo only need to do this step once when you first installed the app.

In the future:

You will still have the Basic Tools folder in your Download folder (or whichever folder you unzipped the zip file into). Please do not delete the unzipped folder (you can delete the zipped file), since it contains the code for Basic Tools in order for the shortcut that you just added onto your Desktop to work.

To install Basic Tools, your computer needs at least 400 MB of space. Since Basic Tools uses Electron, it is not a small app (see why) and will take around 200-370-ish MB of your space. The benefit is that Basic Tools runs completely locally without internet connection, providing security, digital privacy, and a way to not rely everything on internet-based or AI-based tools.

Because Basic Tools runs locally and is isolated from the internet, your app cannot detect newly released updates. To keep up-to-date and enjoy the latest version of Basic Tools, please subscribe to our mailing list here.

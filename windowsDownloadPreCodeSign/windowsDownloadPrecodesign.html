Windows Download Instructions

If you are familiar with downloading a Windows app via a .zip file, unzipping it, finding the Application .exe file in it, creating a shortcut for it on your Desktop, and not deleting the unzipped folder, then you do not need to read this page.

Why This?

I'm adding this instruction page for Windows users is that Basic Tools hasn't been code signed (what is "code sign"?) with Windows yet, meaning, if you download the app directly, it will trigger your Windows firewall and reject the app. This is why when you download the app on the homescreen, it gives you a .zip file that contains the app (along with a bunch of other files that we don't need to ,worry about).

The reason why Basic Tools hasn't been code signed yet (which can make the downloading process much smoother and this instruction page unnecessary) is that it costs a lot of money to code sign with Windows. The cheapest one I found is $129 a year. Capitalists (same with Apple). I will, of course, code sign it as soon as I find enough funding to do so to ensure further security and better user experience. The cost behind code signing is one (sometimes major) hidden cost in software development that is a pain for open-source and non-commercial softwares. If you are interested in supporting Basic Tools, please (please) reach out to me on the "about" page (thanks!).

To smoothly install Basic Tools on Windows:

Step 1:

When you click on the Windows download button on the Basic Tools home screen, you should've received a .zip file in your "Download" folder (or whichever folder that you set your Windows machine to receive downloads to). Right click on it to unzip it (some machines might call this process "Extract All"). If you see that your zipped folder has the words "

:

Step 2:

After unzipping (or extracting) the zip folder, you should see a folder of the same name (but with the "Type" just saying "Folder" and not "Compressed (zipped) Folder"). Enter that unzipped folder.

Step 3:

After you enter the unzipped folder, you should see quite a few files. We do not need to worry about most of them. You just need to find the one that says "BasicTools," with Basic Tools' icon, and has the word "Application" under the document "Type" column:

Step 4:

Right click on the BasicTools application file to show the different options. For some machine, you will see a "Send to" button right away; for some, you need to click on "Show more options" to get to it:

Step 5:

Click on "Show more options" to find "Send to", and then click on the option "Desktop (create shortcut)":

Step 6:

After clicking on "Desktop (create shortcut)," you should see the Basic Tools icon showing up on your desktop. Now, in the future, you can easily access it from there. But, if this is your first time opening Basic Tools or that you have downloaded a new version of it, there's an extra step:

Step 7:

The first time you open Basic Tools, your Windows machine will give you a blue warning window saying: "Windows protected your PC" (because I didn't pay over a hundred dollars for code signing). You can click on "More info" and then "Run Anyway." Yuo only need to do this step once when you first installed the app.

In the future:

You will still have the Basic Tools folder in your Download folder (or whichever folder you unzipped the zip file into). Please do not delete the unzipped folder (you can delete the zipped file), since it contains the code for Basic Tools in order for the shortcut that you just added onto your Desktop to work.
